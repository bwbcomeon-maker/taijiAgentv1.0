# 真实浏览器 QA 检查清单

## 工具优先级

1. 优先使用 Playwright MCP。
2. 其次使用项目已有 Playwright 测试。
3. 再其次使用项目预览环境或 Storybook。
4. 最后才使用代码审查兜底。

如果真实浏览器无法使用，报告中必须写“真实浏览器测试：未验证”，不能写“通过”。

## 检查清单

### 启动项目检查

- 使用项目已有 README、AGENTS.md、package.json scripts 或开发文档启动。
- 不发明不存在的命令。
- 无法启动时记录命令、错误和未验证路径。

### 打开页面检查

- 打开受影响页面。
- 确认页面没有崩溃、白屏或明显控制台错误。
- 确认当前用户上下文、标题、导航位置清楚。

### 默认状态检查

- 检查首屏主内容是否突出。
- 检查主操作是否可见。
- 检查默认数据、空列表或占位状态是否合理。

### 主路径检查

- 执行用户最常见的核心路径。
- 检查每一步是否有状态反馈。
- 检查成功后页面是否保持上下文。

### CRUD 路径检查

- 创建：入口、表单、校验、提交、成功、失败。
- 查看：列表、详情、当前选中项、返回路径。
- 编辑/重命名：入口、保存、取消、失败恢复。
- 删除/归档/清空：入口、确认、取消、成功、失败恢复。

### 表单路径检查

- label、placeholder、帮助文本、错误提示是否清楚。
- 必填、格式错误、禁用、提交中、提交成功和失败状态是否存在。
- 键盘是否能完成填写和提交。

### 列表/表格路径检查

- 扫描线索是否足够。
- 当前项、选中项、hover、focus 是否清楚。
- 长列表滚动是否保持上下文。
- 行操作是否可见、可发现、可访问。

### 搜索/筛选/排序/分页检查

- 搜索入口是否可见。
- 筛选状态是否可理解和可清除。
- 排序方向是否明确。
- 分页、加载更多或虚拟滚动是否不丢上下文。

### 弹窗/抽屉/菜单检查

- 打开和关闭路径是否清楚。
- 焦点是否进入弹窗/菜单。
- Escape 或取消按钮是否可用。
- 关闭后焦点是否回到合理位置。

### 删除/归档/清空等破坏性操作检查

- 入口是否清晰但不易误触。
- 是否有确认、取消、撤销或恢复。
- 失败后是否能恢复或重试。

### 取消/撤销/恢复检查

- 用户能否取消未提交操作。
- 用户能否从错误、误操作或网络失败中恢复。
- 撤销能力不存在时是否有确认和清晰风险提示。

### 空状态检查

- 空状态是否解释原因。
- 是否提供下一步入口。
- 是否避免把空状态误认为加载失败。

### 加载状态检查

- 是否有进行中反馈。
- 是否避免布局跳动。
- 长加载是否有可理解状态。

### 错误状态检查

- 错误信息是否具体、可操作。
- 是否提供重试、返回、恢复或联系支持路径。

### 成功反馈检查

- 操作完成后用户是否知道结果。
- 成功反馈是否不过度打断。

### 禁用状态检查

- 禁用控件是否说明原因。
- 禁用状态是否仍有 accessible name。

### 键盘操作检查

- Tab 顺序符合视觉和任务顺序。
- Enter/Space 能触发按钮或菜单项。
- Escape 能关闭弹窗、菜单或抽屉。
- 键盘用户能完成核心路径。

### 焦点状态检查

- 焦点可见且不刺眼。
- 弹窗打开后焦点进入弹窗。
- 弹窗关闭后焦点回到触发控件或合理位置。

### 响应式检查

- 至少检查常见桌面宽度和窄屏宽度。
- 关键入口不能丢失或被遮挡。
- 文本不能溢出控件。
- 滚动区域必须清楚。

### 截图检查

- 截图关键状态：默认、核心路径、错误、空状态、破坏性确认、窄屏。
- 如果无法截图，报告写“截图：未验证。原因：……”。

### 无法使用浏览器工具时的处理方式

- 明确写出原因，例如无法启动项目、端口不可用、浏览器工具不可用、认证缺失。
- 报告中写“真实浏览器测试：未验证”。
- 只能把代码审查结论写为代码层判断，不能替代浏览器验证。

# Subagent 前端审查模板

复杂 UI、核心路径、大页面重构、PR review、设计系统改动或影响多个页面的前端变更，在可用时使用以下 prompt 启动 subagents。

## 可复制中文 prompt

请作为前端 UX QA 子审查员审查本次前端变更。你只负责下面指定角色，不要替其他角色下结论。请检查代码；如果环境允许，也请通过真实浏览器操作页面。不要因为代码能编译就判断完成。若某项没有执行，必须写“未验证”，不能写“通过”。

审查角色：

1. 功能完整性 QA
2. UX 与信息架构审查
3. 视觉设计与布局审查
4. 可访问性与键盘导航审查
5. 长时间工作体验审查
6. 测试覆盖与回归风险审查

|---|---|---|---|

## 未验证项目

要求：

- 如果代码中实现了用户能力但页面没有可发现 UI 入口，标记为 P1。
- 如果主用户路径无法完成，标记为 P0。
- 如果没有真实浏览器操作，写“真实浏览器测试：未验证”。
- 如果没有截图，写“截图：未验证”。
- 如果没有自动化可访问性检查，写“自动化可访问性：未验证”。

## 六个 subagent 角色

1. 功能完整性 QA：检查能力、API、状态、handler 与 UI 入口是否一致，重点找“实现了但用户找不到入口”。
2. UX 与信息架构审查：检查主内容、辅助内容、高级内容是否分层，用户路径是否符合心智模型。
3. 视觉设计与布局审查：检查层级、间距、对齐、扫描效率、响应式和截图质量。
4. 可访问性与键盘导航审查：检查 accessible name、表单 label、焦点、Tab 顺序、弹窗焦点、Escape/Cancel、键盘路径。
5. 长时间工作体验审查：检查密度、对比、滚动、上下文稳定、疲劳、频繁打断和常用操作可见性。
6. 测试覆盖与回归风险审查：检查 lint/typecheck/test/e2e/browser/screenshot/a11y/visual 回归覆盖和未验证项。

## 每个 subagent 的输出格式

- 审查范围
- 发现的问题
- 严重程度
- 证据
- 建议修复方式
- 是否阻塞完成
- 未验证项目

## 最终汇总格式

等待所有 subagent 返回后才能做最终判断。汇总必须包含：

- 必须修复
- 应该修复
- 可选优化
- 已验证
- 未验证
- 最终状态

最终状态只能是：完成 / 带限制完成 / 未完成 / 阻塞。

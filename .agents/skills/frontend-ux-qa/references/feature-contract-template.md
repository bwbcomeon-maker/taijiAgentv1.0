# 功能契约

| 能力 | 数据/API/状态存在 | UI 入口存在 | 用户反馈存在 | 错误处理存在 | 空/加载/禁用状态 | 键盘/可访问性支持 | E2E/浏览器测试 | 状态 | 备注 |
|---|---:|---:|---:|---:|---:|---:|---:|---|---|

规则：

- 每个用户可感知能力必须有可见、可发现、可访问的入口。
- CRUD 功能必须端到端检查。
- 如果代码支持某个 action，但 UI 不暴露，状态为“失败”。
- 如果入口只存在于隐藏快捷键、未文档化手势、URL hack 或开发者工具中，状态为“失败”。
- 如果检查没有运行，状态为“未验证”。
- 如果存在 P0/P1，则前端任务不能标记为完成。

# 智能体类产品试用评价体系（建议稿）

版本：v0.2  
日期：2026-06-11  
适用范围：面向企业内部真实岗位场景的智能体类产品试用、评估和选型

## 一、编制说明

本评价体系用于指导甲方在试用智能体类产品时，形成统一、可复核、可比较的评价口径。评价重点不放在单轮问答效果、模型参数或厂商演示效果上，而是放在产品能否进入真实岗位流程，把制度、表单、历史经验、业务工具和审批边界转化为可采纳、可追踪、可持续复用的业务成果。

本体系保持技术路线中立，不限定具体厂商、模型、插件、工作流框架或部署形态。参评产品可以采用知识库、工具调用、业务系统接口、场景模板、多智能体协作、本地部署、云端部署或混合部署等不同方式实现，但最终评价以试用结果、业务采纳和安全合规为准。

## 二、评价目标

本评价体系服务于四类目标：

1. **评价真实业务价值**：判断产品是否能在甲方实际岗位任务中减少人工检索、整理、审核、编写和复核成本。
2. **评价流程适配能力**：判断产品是否理解甲方典型业务流程、角色责任、表单字段、风险点和合规边界。
3. **评价可信与可控程度**：判断产品是否能说明依据、标注不确定性、控制幻觉，并保留人工确认边界。
4. **评价落地可持续性**：判断产品是否具备资料接入、场景复用、终端适配、安全合规、诊断运维和持续推广条件。

## 三、评价对象与边界

### 3.1 评价对象

评价对象为参与甲方试用的智能体类产品，包括但不限于企业知识问答、岗位流程辅助、文档生成、表单审核、任务编排、工具调用、业务系统辅助和多步骤工作台类产品。

### 3.2 评价边界

本体系不评价厂商宣传材料中的未试用能力，不以厂商自述功能作为直接得分依据。所有核心得分应来自统一试用任务、统一材料、业务人员真实操作和留存证据。

涉及工作票、操作票、检修计划、缺陷处理、制度问答等具体场景时，应由甲方提供脱敏样例材料，并以甲方现行制度、流程文件和信息安全要求为准。

## 四、评价原则

| 原则 | 含义 | 执行要求 |
| --- | --- | --- |
| 真实场景优先 | 以业务人员高频岗位任务作为主要评价对象 | 避免只做开放式闲聊、通用问答或厂商演示 |
| 统一材料试用 | 所有产品使用同一批脱敏材料和任务说明 | 统一输入、统一周期、统一评分表 |
| 结果证据留存 | 评价结论必须可追溯到试用证据 | 保留输入、输出、依据、修改点、耗时和采纳情况 |
| 业务技术并重 | 技术能力服务于业务闭环 | 不单独用模型参数替代业务成果评价 |
| 安全合规前置 | 试用过程不得突破甲方数据和权限边界 | 涉敏材料必须脱敏，外部调用必须说明边界 |
| 人工责任保留 | 智能体输出不能替代审批、审核和安全责任 | 高风险结论必须保留人工确认提示 |

## 五、评价指标体系

满分 100 分。建议将业务实际效果相关指标设置为 65 分，技术与交付支撑指标设置为 35 分。

| 一级指标 | 权重 | 二级评价重点 | 主要证据 |
| --- | ---: | --- | --- |
| 业务场景理解与适配能力 | 25 | 意图识别、流程节点、角色责任、表单材料、风险合规、澄清能力 | 试用任务输入输出、业务人员评价、缺项识别记录 |
| 任务闭环与产物质量 | 25 | 任务拆解、过程追踪、输出完整性、可编辑性、业务可用性、返工处理 | 阶段记录、最终产物、人工修改点、采纳情况 |
| 可信性、溯源与幻觉控制 | 15 | 依据标注、事实一致性、不确定性表达、幻觉控制、敏感信息处理 | 引用来源、待核验标注、错误记录、风险提示 |
| 知识与工具接入能力 | 10 | 文件资料接入、场景模板复用、工具系统扩展、权限边界 | 导入材料、模板样例、工具调用记录、权限说明 |
| 易用性与业务人员采纳度 | 10 | 上手难度、交互清晰度、语言适配、修改成本、采纳意愿 | 试用人员评分、修改次数、节省时间、反馈记录 |
| 部署适配、安全合规与运维 | 10 | 终端环境、数据安全、权限审计、故障诊断、运维持续性 | 部署说明、安全说明、日志审计、诊断材料 |
| 性能、稳定性与成本 | 5 | 响应效率、长任务稳定性、失败恢复、使用成本 | 任务耗时、异常记录、恢复情况、成本估算 |

## 六、指标定义与评分口径

### 6.1 业务场景理解与适配能力（25 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 业务意图识别 | 5 | 能准确识别用户要完成的业务目标，而不是只按字面回答问题 |
| 流程节点理解 | 5 | 能理解任务所处阶段、前后置条件、流转节点和审批边界 |
| 角色责任识别 | 4 | 能区分发起人、审核人、执行人、监护人、管理人员等不同职责 |
| 表单与材料理解 | 4 | 能正确理解样例表单、制度条款、历史案例和附件材料之间的关系 |
| 风险与合规意识 | 5 | 能识别安全风险、合规风险、缺项漏项和不确定内容 |
| 澄清与边界控制 | 2 | 信息不足时主动澄清或标注边界，避免直接编造 |

高分特征：能把业务材料转化为流程化判断，准确识别角色、字段、风险和依据。  
低分特征：只做文本概括或通用建议，不能识别流程责任和安全合规边界。

### 6.2 任务闭环与产物质量（25 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 多步骤任务拆解 | 5 | 能把复杂任务拆成可执行步骤，并说明当前进度和下一步 |
| 过程可追踪 | 4 | 能保留关键过程、判断依据、阶段状态和用户确认点 |
| 输出完整性 | 6 | 产物覆盖任务要求，不遗漏关键字段、风险、责任或结论 |
| 输出可编辑性 | 4 | 产物格式清晰，便于复制、修改、归档或转成正式文档 |
| 业务可用性 | 4 | 产物可直接或少量修改后用于业务办理、审核、沟通、汇报或整改 |
| 异常与返工处理 | 2 | 任务失败、信息缺失或用户调整要求时，能说明原因并继续处理 |

高分特征：形成可复核产物，业务人员修改量少，并能继续推进下一步。  
低分特征：回答流畅但不可直接使用，缺少过程记录或无法处理返工。

### 6.3 可信性、溯源与幻觉控制（15 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 引用与依据标注 | 4 | 引用制度、规程、历史材料时说明来源、条款位置或材料出处 |
| 事实一致性 | 4 | 输出与甲方提供材料一致，不混用口径、不张冠李戴 |
| 不确定性表达 | 3 | 对缺失信息、过期标准、无法确认事项标注待核验 |
| 幻觉控制 | 3 | 不编造政策、流程、设备状态、数据、审批结论或内部规则 |
| 敏感信息处理 | 1 | 不在输出中暴露不应公开的内部信息或敏感字段 |

高分特征：关键结论有依据，缺口表达清楚，高风险内容保留人工确认。  
低分特征：编造条款、夸大确定性，或把推测写成事实。

### 6.4 知识与工具接入能力（10 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 文件与资料接入 | 3 | 支持制度文件、表单模板、历史案例、附件材料等资料导入和理解 |
| 场景模板复用 | 3 | 能沉淀可复用的任务模板、提示规则、流程步骤或检查清单 |
| 工具与系统扩展 | 2 | 具备与本地工具、业务系统、知识库或流程系统对接的扩展能力 |
| 权限与边界控制 | 2 | 工具调用、资料访问和结果输出具备权限控制和人工确认机制 |

本项采用能力结果口径，不限定具体实现方式。参评产品可通过知识库、模板、工具编排、系统接口或其他方式实现。

### 6.5 易用性与业务人员采纳度（10 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 上手难度 | 2 | 非技术人员能在简短引导后独立完成任务 |
| 交互清晰度 | 2 | 产品能清楚说明需要用户补充什么、已完成什么、下一步是什么 |
| 语言与文风适配 | 2 | 输出符合甲方业务语境、公文表达和部门沟通习惯 |
| 修改成本 | 2 | 业务人员二次修改量明显减少 |
| 采纳反馈 | 2 | 试用人员愿意在真实工作中继续使用相关能力 |

建议记录试用人员主观评分、人工修改次数、修改耗时、最终采纳情况和典型反馈。

### 6.6 部署适配、安全合规与运维（10 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 终端环境适配 | 2 | 符合甲方办公终端、浏览器、操作系统或国产化环境要求 |
| 数据安全 | 2 | 能说明数据存储、传输、调用外部模型和敏感资料处理方式 |
| 权限与审计 | 2 | 具备用户权限、操作记录、日志留存和必要审计能力 |
| 故障诊断 | 2 | 出现异常时便于定位问题、导出诊断信息和恢复使用 |
| 运维可持续性 | 2 | 具备安装、升级、配置、备份、停用和支持服务方案 |

该项应根据甲方信息安全、终端管理和运维制度执行，不限定云端、本地或混合部署形态，但最终方案必须满足甲方数据安全与终端运行要求。

### 6.7 性能、稳定性与成本（5 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 响应效率 | 1.5 | 常规任务响应时间可接受 |
| 长任务稳定性 | 1.5 | 多步骤、长文本、附件类任务能稳定完成 |
| 失败恢复 | 1 | 网络、模型、文件或系统异常后能提示原因并继续处理 |
| 使用成本 | 1 | 模型、算力、部署、运维和培训成本合理 |

## 七、评价实施流程

| 阶段 | 主要动作 | 输出物 |
| --- | --- | --- |
| 准备阶段 | 确定参评产品、试用人员、试用周期和脱敏材料 | 试用方案、任务包、评分表 |
| 试用阶段 | 业务人员按统一任务使用产品，并保留过程证据 | 原始输入、产品输出、引用依据、耗时记录 |
| 评分阶段 | 按指标体系分别由业务、信息化/安全、运维人员评分 | 分项评分表、问题清单 |
| 复核阶段 | 对关键扣分项、重大风险项和得分接近项进行复核 | 复核意见、风险说明 |
| 结论阶段 | 形成产品适用场景、落地条件和后续建议 | 试用评价报告、选型建议 |

建议试用周期为 7 至 14 天。厂商可提供必要安装、配置和使用引导，但不应代替甲方完成评分任务。

## 八、试用任务包设计

每个参评产品至少完成 5 类任务。甲方可根据实际业务选择任务，但各产品应使用同一任务说明和同一批脱敏材料。

| 任务类型 | 目标产物 | 重点观察 |
| --- | --- | --- |
| 制度问答与溯源 | 带依据的问答结果 | 是否引用准确、边界清楚、不编造 |
| 流程表单审核 | 审核意见、缺项清单、风险提示 | 是否识别字段、角色、流程和安全风险 |
| 操作步骤校核 | 步骤校核意见、危险点提示、确认清单 | 是否理解顺序、前置条件和人工确认边界 |
| 计划编制 | 工作计划、人员分工、材料清单、风险措施 | 是否能形成可执行计划 |
| 缺陷或问题闭环 | 分级建议、处置流程、整改清单、复盘材料 | 是否能从描述推进到闭环处置 |
| 综合材料输出 | 汇报、纪要、报告初稿或整改说明 | 是否符合甲方语言习惯和正式材料要求 |

## 九、证据采集与计分方法

### 9.1 证据采集

每项任务至少留存以下证据：

1. 原始输入材料和任务说明。
2. 产品原始输出。
3. 产品引用的依据或来源。
4. 任务耗时。
5. 业务人员人工修改点。
6. 最终是否采纳。
7. 试用人员评分和简要评价。

### 9.2 评分主体建议

| 评分主体 | 建议权重 | 重点职责 |
| --- | ---: | --- |
| 业务部门评分人员 | 70% | 评价任务完成质量、业务可用性、修改成本和采纳意愿 |
| 信息化/安全评分人员 | 20% | 评价系统接入、部署适配、数据安全、权限和审计 |
| 运维管理评分人员 | 10% | 评价安装配置、稳定性、故障诊断、升级维护和支持能力 |

### 9.3 评分等级建议

| 等级 | 分数区间 | 建议解释 |
| --- | ---: | --- |
| A | 90-100 | 可优先进入试点扩大或采购论证 |
| B | 80-89 | 具备落地价值，需补齐部分场景、合规或运维条件 |
| C | 70-79 | 可在有限场景继续观察，不宜直接大范围推广 |
| D | 60-69 | 能力或落地条件不足，需完成明显改进后再评估 |
| E | 60 以下 | 暂不建议进入正式试点或选型 |

## 十、结果应用建议

最终结论不宜只给总分排名，应同时给出：

1. 各一级指标得分和主要扣分项。
2. 产品适合优先落地的业务场景。
3. 不适合或需谨慎使用的业务边界。
4. 数据安全、部署运维和系统接入前置条件。
5. 后续试点、扩围、采购或淘汰建议。

对于得分接近的产品，建议优先比较真实任务采纳率、人工修改成本、可信性问题数量和安全合规风险，而不是单纯比较回答速度或界面观感。

## 十一、中立措辞要求

为保证评价体系公开、公平、可解释，建议避免使用指向特定厂商或特定实现方式的表述。

| 不建议表述 | 建议表述 |
| --- | --- |
| 必须具备专家团 | 具备多步骤任务协同和过程可追踪能力 |
| 必须支持某某 Skill | 具备可复用场景模板或任务能力沉淀机制 |
| 必须支持某某 MCP | 具备可扩展工具或业务系统接入能力 |
| 必须本地桌面安装 | 符合甲方终端环境与数据安全要求的部署方式 |
| 必须支持工作票、操作票 | 能适配甲方典型流程表单和作业类业务场景 |

## 十三、状态边界

本文件为评价体系建议稿。涉及具体制度、流程、表单、系统接口、数据分类分级和安全要求时，应以甲方现行制度、脱敏样例材料和信息安全要求为准。

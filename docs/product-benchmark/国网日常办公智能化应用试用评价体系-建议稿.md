# 国网日常办公智能化应用试用评价体系（建议稿）

版本：v0.5  
日期：2026-06-12  
适用范围：日常办公与典型业务流程智能化应用试用、评估和选型

## 一、编制说明

本评价体系用于指导甲方在试用日常办公智能化应用时，形成统一、可复核、可比较的评价口径。评价重点不放在单轮问答效果、模型参数或厂商演示效果上，而是放在产品能否围绕真实办公任务和典型业务流程，把制度、材料、表单、流程和审批边界转化为可采纳、可复核、可持续复用的业务成果。

本体系不限定具体厂商、模型、部署方式或实现框架。参评产品可以通过知识库、业务能力模板、工具接入、流程编排、本地化交付、云端服务或混合方式实现，但最终评价以统一试用任务、业务人员真实操作、输出产物质量、量化过程证据和运行保障结果为准。

本版本根据领导反馈，采用“一级指标相对中性、二级指标体现业务能力”的设计方式。评价体系不把某类业务场景直接写成一级指标，也不把某厂商实现方式写成门槛；但会在二级指标、评分口径和任务包中明确考察日常办公文档、典型业务表单、流程节点、业务能力沉淀和多步骤任务闭环等实际能力。

## 二、评价目标

评价日常办公业务办理与文档产出能力：判断产品是否能辅助完成制度问答、会议纪要、汇报材料、请示方案、整改清单等高频办公任务，并形成符合甲方日常使用习惯的文档或表格产物。

评价典型业务表单与流程适配能力：判断产品是否能理解甲方指定的一单两票、检修计划、缺陷记录等典型表单材料，识别字段、角色、流程节点、风险点和人工确认边界。

评价可复用业务能力沉淀能力：判断产品是否能把高频任务沉淀为可复用的业务模板、检查清单、输出格式和任务入口，减少每次从零开始配置或提示的成本。

评价多步骤任务闭环与过程留痕能力：判断产品是否能把复杂任务拆解为阶段、记录过程、沉淀产物，并在信息不足、口径变化或任务返工时继续推进。

评价依据引用与风险边界表达能力：判断产品是否能基于甲方提供材料作答，说明依据来源，标注待核验事项，并在涉及审批、安全、责任认定等场景时保留人工确认边界。

评价真实采纳和效率提升能力：判断业务人员是否愿意使用，是否减少检索、整理、编写、审核和二次修改成本，并通过统一任务下的耗时、一次可用率和失败重试情况形成可比较证据。

## 三、评价对象与边界

评价对象为参与甲方试用的日常办公智能化应用，包括企业知识问答、业务材料生成、流程表单辅助、任务办理辅助、工具调用、文档整理和多步骤工作台类产品。

本体系不评价厂商宣传材料中的未试用能力，不以厂商自述功能作为直接得分依据。所有核心得分应来自统一材料、统一任务、业务人员真实操作和留存证据。

涉及一单两票、检修计划、缺陷处理、制度问答等具体场景时，应由甲方提供脱敏样例材料，并以甲方现行制度、流程文件、终端环境和管理要求为准。产品输出用于辅助分析、材料生成和风险提示，不替代审批、审核和安全责任。

效率、稳定性和成本类指标应在输出质量达标的前提下评价。生成速度快但结果不可用、依据不清或需要大量返工的，不应仅因响应快而获得高分。

## 四、评价原则

| 原则 | 含义 | 执行要求 |
| --- | --- | --- |
| 业务任务优先 | 以业务人员真实任务完成质量作为主要评价对象 | 避免只看开放式闲聊、模型参数或厂商演示 |
| 统一材料试用 | 所有产品使用同一批脱敏材料和任务说明 | 统一输入、统一周期、统一评分表 |
| 产物可采纳 | 评价输出能否直接或少量修改后进入业务使用 | 保留原始输出、修改点、采纳情况和节省时间 |
| 依据可核验 | 关键结论应说明材料来源或制度依据 | 无法确认事项必须标注待核验 |
| 量化口径一致 | 耗时、修改次数、一次通过和失败重试等数据应按同一口径采集 | 同一任务、同一材料、同一人员或同类人员条件下比较 |
| 运行边界清楚 | 试用过程应符合甲方既有终端、网络环境和安全管控要求 | 涉敏材料脱敏，外部调用和资料留存边界清楚 |
| 人工责任保留 | 产品输出不能替代审批、审核和安全责任 | 高风险结论必须保留人工确认提示 |

## 五、评价指标体系

满分 100 分。建议将日常办公、典型表单、业务能力沉淀和任务闭环作为主体，将依据边界、采纳效率、运行保障和成本可控作为支撑指标，避免评价被具体实现方式或模型参数牵引。

下表为一级指标总览，第六章按本表顺序逐项展开；二级指标名称、分值和评分口径以第六章为准。

| 一级指标 | 权重 | 二级评价重点 | 主要证据 |
| --- | ---: | --- | --- |
| 日常办公业务办理与文档产出能力 | 20 | 制度问答、会议纪要、汇报材料、请示方案、整改清单、表格清单产物 | 办公任务输入输出、文档初稿、业务人员修改点、采纳情况 |
| 典型业务表单与流程适配能力 | 20 | 工作票、操作票、检修计划、缺陷处理、角色责任、审批边界 | 样例表单辅助检查意见、缺项清单、风险提示、人工确认记录 |
| 可复用业务能力沉淀能力 | 15 | 任务模板、检查清单、输出格式、业务规则复用、示例任务入口 | 业务能力模板、场景样例、复用记录、配置说明 |
| 多步骤任务闭环与过程留痕能力 | 15 | 任务拆解、阶段状态、过程记录、产物清单、返工处理、后续建议 | 阶段记录、最终产物、用户确认点、重做记录 |
| 依据引用与风险边界表达能力 | 12 | 依据标注、事实一致、待核验表达、风险提示、人工确认边界 | 引用来源、待核验事项、风险提示、复核记录 |
| 易用性与业务人员采纳度 | 8 | 上手难度、交互清晰度、业务语言、修改成本、采纳反馈 | 试用人员评分、节省时间、满意度和采纳反馈 |
| 试用效率、稳定性与成本可控能力 | 5 | 任务有效完成时间、一次可用率、失败与重试情况、资源与使用成本 | 任务计时记录、一次通过记录、失败重试记录、成本说明 |
| 实施适配与运行保障能力 | 5 | 终端环境适配、资料处理边界、权限与留痕、诊断恢复、运维支持 | 实施说明、运行记录、诊断材料、支持记录 |

## 六、指标定义与评分口径

### 6.1 日常办公业务办理与文档产出能力（20 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 制度问答与依据检索 | 3 | 能根据制度、规程、管理办法回答具体问题，并说明依据或材料位置 |
| 会议纪要与记录整理 | 3 | 能从会议记录、录音转写或零散材料中整理纪要、待办和责任事项 |
| 汇报与报告材料生成 | 4 | 能生成结构清楚、表达稳妥、便于修改的汇报、总结、报告初稿 |
| 请示方案与说明材料 | 3 | 能围绕背景、依据、事项、风险和建议形成规范化说明材料 |
| 整改清单与跟踪事项 | 3 | 能从问题描述生成整改措施、责任分工、时限和复盘要点 |
| 表格与清单产物 | 4 | 能生成计划表、问题清单、责任矩阵、材料清单等可编辑表格 |

高分特征：输出接近甲方日常办公材料形态，标题层级、段落表达、表格字段和责任事项清楚，业务人员少量修改即可使用。

低分特征：只给泛泛回答或普通聊天式建议，不能形成纪要、汇报、请示、方案、整改清单等可交付产物。

主要留证材料：原始输入、产品输出、人工修改稿、采纳情况、修改次数和修改耗时。

### 6.2 典型业务表单与流程适配能力（20 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 典型表单识别与理解 | 4 | 能理解甲方指定的一单两票、检修计划、缺陷记录等典型表单材料 |
| 字段完整性校核 | 4 | 能识别字段缺失、填写不一致、材料不完整和关键附件缺漏 |
| 角色责任识别 | 3 | 能区分发起、审核、执行、监护、管理等不同角色职责 |
| 流程节点与前后置条件 | 4 | 能判断任务所处阶段、前置条件、流转节点和审批边界 |
| 风险与缺项提示 | 4 | 能识别安全风险、合规风险、缺项漏项和不确定事项 |
| 人工确认边界 | 1 | 涉及审批、安全、责任认定时保留人工确认提示 |

高分特征：能把表单、制度、附件和历史材料关联起来，形成缺项清单、风险提示、修改建议和人工确认项，而不是只做文字润色。

低分特征：只识别表面文本错误，不能看出字段缺漏、角色责任、流程前后置条件或风险边界。

主要留证材料：样例表单、检查意见、缺项清单、风险提示、人工确认记录。

### 6.3 可复用业务能力沉淀能力（15 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 高频任务模板化 | 4 | 能将制度问答、材料生成、表单辅助检查等高频任务沉淀为可复用模板 |
| 检查清单沉淀 | 3 | 能将字段完整性、风险点、责任分工、待核验事项沉淀为检查清单 |
| 输出格式复用 | 3 | 能复用纪要、汇报、请示、整改清单、审核意见等常用输出格式 |
| 业务规则复用 | 3 | 能复用甲方提供的制度口径、流程节点、角色责任和材料要求 |
| 示例任务入口 | 2 | 能为业务人员提供清晰的示例任务或任务入口，降低重复说明成本 |

高分特征：同类任务再次试用时，不需要业务人员反复解释格式、字段和检查口径，产品能沿用已沉淀的任务模板和输出规范。

低分特征：每次任务都依赖临时提示，无法沉淀规则、清单和输出格式，换一个人员或任务就需要重新组织。

主要留证材料：业务能力模板、检查清单、示例任务、复用前后对比、配置说明。

### 6.4 多步骤任务闭环与过程留痕能力（15 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 任务拆解 | 3 | 能把复杂任务拆成可执行步骤，并说明当前进度和下一步 |
| 阶段状态与过程记录 | 3 | 能保留关键输入、判断依据、阶段状态和用户确认点 |
| 产物清单与版本留存 | 3 | 能形成可编辑、可复核、可归档的阶段产物和最终产物 |
| 返工处理 | 3 | 信息缺失、用户改口径或任务失败时能说明原因并继续处理 |
| 后续办理建议 | 3 | 能给出后续办理、复核、整改、归档或复盘建议 |

高分特征：产品不仅给一次性答案，还能持续推进任务，保留过程、产物和确认点，便于业务人员复核和接续办理。

低分特征：回答结束后没有阶段记录、没有产物清单，用户改口径或补材料后需要重新开始。

主要留证材料：阶段记录、产物清单、最终文件、用户确认点、返工记录。

### 6.5 依据引用与风险边界表达能力（12 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 引用与依据标注 | 3 | 引用制度、规程、历史材料时说明来源、条款位置或材料出处 |
| 事实一致性 | 3 | 输出与甲方提供材料一致，不混用口径、不张冠李戴 |
| 待核验表达 | 2 | 对缺失信息、过期标准、无法确认事项标注待核验 |
| 风险提示 | 2 | 对安全、合规、责任、材料缺失等风险作出审慎提示 |
| 人工确认边界 | 2 | 不替代审批、审核、安全确认和责任认定，必要时提示人工复核 |

高分特征：关键结论能回到材料依据，风险和不确定事项说清楚，高风险场景不越过人工确认边界。

低分特征：输出听起来完整但依据不清，或把缺失信息、推测判断写成确定结论。

主要留证材料：引用来源、待核验事项、风险提示、人工复核记录。

### 6.6 易用性与业务人员采纳度（8 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 上手难度 | 2 | 非技术人员能在简短引导后独立完成任务 |
| 交互清晰度 | 1 | 产品能清楚说明需要用户补充什么、已完成什么、下一步是什么 |
| 业务语言适配 | 2 | 输出符合甲方业务语境、公文表达和部门沟通习惯 |
| 修改成本 | 2 | 业务人员二次修改量明显减少 |
| 采纳反馈 | 1 | 试用人员愿意在真实工作中继续使用相关能力 |

高分特征：业务人员不需要理解模型、插件或技术概念，能按日常工作任务直接使用，并愿意继续用于真实办公。

低分特征：操作入口复杂，必须依赖技术人员解释，输出语言不符合甲方业务表达，修改成本仍然较高。

主要留证材料：试用人员评分、修改次数、修改耗时、满意度、采纳反馈。

### 6.7 试用效率、稳定性与成本可控能力（5 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 任务有效完成时间 | 2 | 在输出质量达标前提下，比较完成同一任务所需时间，包括生成时间和必要交互时间 |
| 一次可用率 | 1 | 同一任务首次完整输出是否达到少量修改可用，避免只看回答速度 |
| 失败与重试情况 | 1 | 统计任务中断、输出为空、格式错误、无法继续处理或必须重开任务等情况 |
| 资源与使用成本 | 1 | 评估模型、算力、部署、运维和培训成本是否可接受 |

高分特征：在统一任务、统一材料、统一评分口径下，产品能稳定完成任务，返工少，一次可用率较高。

低分特征：响应快但不可用，或频繁中断、重试、格式错乱，需要业务人员反复补救。

主要留证材料：任务计时记录、一次通过记录、失败重试记录、人工修改耗时、成本说明。

### 6.8 实施适配与运行保障能力（5 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 终端环境适配 | 1 | 能与甲方既有办公终端、浏览器、操作系统和桌面管控要求兼容 |
| 资料处理边界 | 1 | 能说明资料导入、存储、传输、外部调用和留存边界 |
| 权限与留痕 | 1 | 具备必要的用户权限、操作记录、日志留存或审计支持 |
| 诊断与恢复 | 1 | 出现异常时便于定位问题、导出诊断信息和恢复使用 |
| 运维支持 | 1 | 具备安装、升级、配置、备份、停用和支持服务方案 |

高分特征：产品能在甲方既有办公和管控条件下试用，问题可诊断、可恢复，运行边界说明清楚。

低分特征：对终端环境、资料处理、权限留痕和异常恢复缺少说明，试用中出现问题后难以定位。

主要留证材料：实施说明、运行记录、诊断材料、支持记录。

## 七、试用任务包设计

每个参评产品至少完成 5 类任务。甲方可根据实际业务选择任务，但各产品应使用同一任务说明和同一批脱敏材料，并按统一口径记录任务耗时、一次通过、失败重试和人工修改情况。

| 任务类型 | 目标产物 | 重点观察 |
| --- | --- | --- |
| 日常办公文档生成 | 纪要、汇报、请示、方案、整改清单 | 是否结构规范、语气合适、少改可用，人工修改耗时是否减少 |
| 制度问答与依据引用 | 带依据的问答结果、待核验事项 | 是否引用准确、边界清楚，是否能在合理时间内完成 |
| 典型业务表单辅助检查 | 一单两票或甲方指定表单的检查意见、缺项清单、风险提示 | 是否识别字段、角色、流程和风险，是否保留人工确认边界 |
| 操作步骤或办理流程校核 | 步骤校核意见、确认清单、危险点提示 | 是否理解顺序、前置条件和人工确认边界，是否出现遗漏或误判 |
| 检修计划与任务分解 | 计划初稿、人员分工、材料清单、风险措施 | 是否能形成可执行计划，是否能沉淀阶段产物 |
| 缺陷或问题闭环与复盘材料 | 分级建议、处置流程、整改清单、复盘材料 | 是否能从描述推进到闭环处置，失败或信息不足时能否继续处理 |

## 八、评价实施流程

建议试用周期为 7 至 14 天。厂商可提供必要安装、配置和使用引导，但不应代替甲方完成评分任务。

| 阶段 | 主要动作 | 输出物 |
| --- | --- | --- |
| 准备阶段 | 确定参评产品、试用人员、周期、脱敏材料、统一计时口径和评分表 | 试用方案、任务包、评分表、过程记录表 |
| 试用阶段 | 业务人员按统一任务使用产品，并保留过程证据和量化记录 | 原始输入、产品输出、依据、耗时记录、失败重试记录 |
| 评分阶段 | 业务、信息化/安全、运维人员分别评分 | 分项评分表、问题清单、量化统计表 |
| 复核阶段 | 对关键扣分项、重大风险项、得分接近项和异常耗时项复核 | 复核意见、风险说明、异常说明 |
| 结论阶段 | 形成适用场景、落地条件和后续建议 | 试用评价报告、选型建议 |

## 九、证据采集与计分方法

### 9.1 证据采集

每项任务至少留存以下证据：

1. 原始输入材料和任务说明。
2. 产品原始输出。
3. 产品引用的依据或来源。
4. 任务开始时间、首次输出时间和可采纳完成时间。
5. 业务人员人工修改点、修改次数和修改耗时。
6. 最终是否采纳，首次输出是否一次通过。
7. 是否发生失败、重试、中断、空输出或明显格式错乱。
8. 试用人员评分和简要评价。

### 9.2 统一计时与量化记录口径

| 量化字段 | 记录口径 |
| --- | --- |
| 任务开始时间 | 用户提交完整任务材料并发起任务的时间 |
| 首次输出时间 | 产品首次给出完整可阅读结果的时间 |
| 可采纳完成时间 | 输出达到少量修改可用的时间 |
| 人工修改次数 | 业务人员为形成最终可用产物进行的实质性修改轮次 |
| 人工修改耗时 | 业务人员从产品输出到最终可用稿的处理时间 |
| 是否一次通过 | 首次完整输出是否基本可用，且无需重新发起任务 |
| 是否失败或重试 | 是否出现中断、卡死、空输出、明显格式错乱或必须重新开始 |

### 9.3 评分主体建议

| 评分主体 | 建议权重 | 重点职责 |
| --- | ---: | --- |
| 业务部门评分人员 | 70% | 评价任务完成质量、产物可用性、修改成本和采纳意愿 |
| 信息化/安全评分人员 | 20% | 评价资料接入、运行边界、权限留痕和实施适配 |
| 运维管理评分人员 | 10% | 评价安装配置、稳定性、诊断恢复、升级维护和支持能力 |

### 9.4 评分等级建议

| 等级 | 分数区间 | 建议解释 |
| --- | ---: | --- |
| A | 90-100 | 可优先进入试点扩大或采购论证 |
| B | 80-89 | 具备落地价值，需补齐部分场景、运行保障或运维条件 |
| C | 70-79 | 可在有限场景继续观察，不宜直接大范围推广 |
| D | 60-69 | 能力或落地条件不足，需完成明显改进后再评估 |
| E | 60 以下 | 暂不建议进入正式试点或选型 |

## 十、结果应用建议

最终结论不宜只给总分排名，应同时给出：

1. 各一级指标得分和主要扣分项。
2. 产品适合优先落地的日常办公和业务办理场景。
3. 不适合或需谨慎使用的业务边界。
4. 资料处理、运行保障和系统接入前置条件。
5. 有效完成时间、人工修改成本、失败重试情况和一次可用率。
6. 后续试点、扩围、采购或淘汰建议。

对于得分接近的产品，建议优先比较真实任务采纳率、人工修改成本、依据错误数量、失败重试情况和有效完成时间，而不是单纯比较回答速度、模型参数或界面观感。

## 十一、能力导向措辞要求

为保证评价体系公开、公平、可解释，建议避免使用指向特定厂商或特定实现方式的表述。

| 不建议表述 | 建议表述 |
| --- | --- |
| 必须具备专家团 | 具备多步骤任务协同和过程留痕能力 |
| 必须支持某某插件或协议 | 具备可扩展工具或业务系统协同能力 |
| 必须本地安装 | 符合甲方既有终端和运行管理要求的交付方式 |
| 将一单两票写成准入条件 | 能适配甲方指定的典型业务表单和流程材料 |
| 必须具备某厂商功能 | 具备对应业务能力或可复用任务模板 |
| 响应速度最快 | 在输出质量达标前提下，有效完成时间更短、返工更少 |

## 十四、状态边界

本文件为评价体系建议稿。涉及具体制度、流程、表单、系统接口、数据分类分级和运行管理要求时，应以甲方现行制度、脱敏样例材料和管理要求为准。效率、稳定性和成本相关指标应结合任务难度、材料完整性、终端环境和试用人员熟悉程度综合判断，不作为脱离业务产物质量的单独排名依据。

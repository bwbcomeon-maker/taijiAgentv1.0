# 国网日常办公智能化应用试用评价体系（建议稿）

版本：v0.3  
日期：2026-06-12  
适用范围：日常办公与典型业务流程智能化应用试用、评估和选型

## 一、编制说明

本评价体系用于指导甲方在试用日常办公智能化应用时，形成统一、可复核、可比较的评价口径。评价重点不放在单轮问答效果、模型参数或厂商演示效果上，而是放在产品能否进入真实办公和业务办理场景，把制度、材料、表单、流程和审批边界转化为可采纳、可复核、可持续复用的业务成果。

本体系保持技术路线中立，不限定具体厂商、模型、部署方式或实现框架。参评产品可以通过知识库、业务技能、能力模板、工具接入、流程编排、本地化交付、云端服务或混合方式实现，但最终评价以统一试用任务、业务人员真实操作、输出产物质量和安全合规结果为准。

本版本根据领导反馈，进一步弱化技术实现描述，强化日常办公业务、典型业务表单、办公文档格式、可信依据和功能竞争力评价。

## 二、评价目标

评价日常办公业务办理能力：判断产品是否能辅助完成制度问答、会议纪要、汇报材料、请示方案、整改清单等高频办公任务。

评价典型业务流程适配能力：判断产品是否理解作业类表单、流程节点、角色责任、风险点和人工确认边界。

评价办公文档产物质量：判断产品输出是否符合甲方常用文档格式、公文表达、表格结构和归档修改要求。

评价可信与可控程度：判断产品是否能基于材料作答、说明依据、标注待核验事项，并避免把推测写成事实。

评价真实采纳价值：判断业务人员是否愿意使用，是否减少检索、整理、编写、审核和二次修改成本。

## 三、评价对象与边界

评价对象为参与甲方试用的日常办公智能化应用，包括企业知识问答、业务材料生成、流程表单审核、任务办理辅助、工具调用、文档整理和多步骤工作台类产品。

本体系不评价厂商宣传材料中的未试用能力，不以厂商自述功能作为直接得分依据。所有核心得分应来自统一材料、统一任务、业务人员真实操作和留存证据。

涉及工作票、操作票、检修计划、缺陷处理、制度问答等具体场景时，应由甲方提供脱敏样例材料，并以甲方现行制度、流程文件、终端环境和信息安全要求为准。

## 四、评价原则

| 原则 | 含义 | 执行要求 |
| --- | --- | --- |
| 功能可用优先 | 以业务人员真实任务完成质量作为主要评价对象 | 避免只看开放式闲聊、模型参数或厂商演示 |
| 统一材料试用 | 所有产品使用同一批脱敏材料和任务说明 | 统一输入、统一周期、统一评分表 |
| 产物可采纳 | 评价输出能否直接或少量修改后进入业务使用 | 保留原始输出、修改点、采纳情况和节省时间 |
| 依据可核验 | 关键结论应说明材料来源或制度依据 | 无法确认事项必须标注待核验 |
| 安全边界清楚 | 试用过程不得突破甲方数据和权限边界 | 涉敏材料脱敏，外部调用和资料留存边界清楚 |
| 人工责任保留 | 产品输出不能替代审批、审核和安全责任 | 高风险结论必须保留人工确认提示 |

## 五、评价指标体系

满分 100 分。建议将功能可用、业务产物和真实采纳相关指标作为主体，将安全合规与交付适配作为基础门槛。

| 一级指标 | 权重 | 二级评价重点 | 主要证据 |
| --- | ---: | --- | --- |
| 日常办公业务办理能力 | 20 | 制度问答、会议纪要、汇报材料、请示方案、整改清单、流程辅助 | 办公任务输入输出、业务人员修改点、采纳情况 |
| 典型业务表单与流程适配能力 | 20 | 工作票、操作票、检修计划、缺陷处理、角色责任、审批边界 | 样例表单审核意见、缺项清单、风险提示 |
| 办公文档生成与格式规范能力 | 15 | 标题层级、表格格式、公文语气、Word 风格产物、可编辑性 | 文档初稿、格式检查、人工修改记录 |
| 可信依据与编造控制能力 | 15 | 依据标注、事实一致性、待核验表达、编造控制、敏感信息处理 | 引用来源、错误记录、待核验事项、风险提示 |
| 任务闭环与过程可追踪能力 | 15 | 任务拆解、阶段状态、过程记录、产物清单、返工处理 | 阶段记录、最终产物、用户确认点、重做记录 |
| 易用性与业务人员采纳度 | 10 | 上手难度、交互清晰度、业务语言、修改成本、使用意愿 | 试用人员评分、节省时间、满意度和采纳反馈 |
| 安全合规与交付适配能力 | 5 | 终端环境、数据安全、权限边界、诊断支持、运维可持续性 | 安全说明、部署说明、诊断材料、运维记录 |

## 六、指标定义与评分口径

### 5.1 日常办公业务办理能力（20 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 制度问答与政策检索 | 4 | 能根据制度、规程、管理办法回答具体问题，并说明依据或材料位置 |
| 会议纪要与记录整理 | 3 | 能从会议记录、录音转写或零散材料中整理纪要、待办和责任事项 |
| 汇报与报告材料生成 | 4 | 能生成结构清楚、表达稳妥、便于修改的汇报、总结、报告初稿 |
| 请示方案与说明材料 | 3 | 能围绕背景、依据、事项、风险和建议形成规范化说明材料 |
| 整改清单与跟踪事项 | 3 | 能从问题描述生成整改措施、责任分工、时限和复盘要点 |
| 办公流程辅助 | 3 | 能提示下一步办理事项、需补材料和人工确认点 |

### 5.2 典型业务表单与流程适配能力（20 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 流程表单理解 | 4 | 能理解样例表单、制度条款、历史案例和附件材料之间的关系 |
| 字段完整性校核 | 4 | 能识别字段缺失、填写不一致、材料不完整和关键附件缺漏 |
| 角色责任识别 | 3 | 能区分发起、审核、执行、监护、管理等不同角色职责 |
| 流程节点与前后置条件 | 4 | 能判断任务所处阶段、前置条件、流转节点和审批边界 |
| 风险与合规提示 | 4 | 能识别安全风险、合规风险、缺项漏项和不确定事项 |
| 人工确认边界 | 1 | 涉及审批、安全、责任认定时保留人工确认提示 |

### 5.3 办公文档生成与格式规范能力（15 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 文档结构完整 | 3 | 输出包含标题、背景、依据、正文、结论或下一步等必要结构 |
| 格式规范性 | 3 | 标题层级、段落缩进、表格字段、编号和版面适合进入正式文档 |
| 公文与业务语气 | 3 | 表达符合甲方日常办公、公文材料和部门沟通习惯 |
| 表格与清单产物 | 3 | 能生成计划表、问题清单、整改表、责任矩阵等可编辑表格 |
| 少量修改可用 | 3 | 业务人员少量修改后即可用于汇报、沟通、归档或办理 |

### 5.4 可信依据与编造控制能力（15 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 引用与依据标注 | 4 | 引用制度、规程、历史材料时说明来源、条款位置或材料出处 |
| 事实一致性 | 4 | 输出与甲方提供材料一致，不混用口径、不张冠李戴 |
| 待核验表达 | 3 | 对缺失信息、过期标准、无法确认事项标注待核验 |
| 编造控制 | 3 | 不编造政策、流程、设备状态、数据、审批结论或内部规则 |
| 敏感信息处理 | 1 | 不在输出中暴露不应公开的内部信息或敏感字段 |

### 5.5 任务闭环与过程可追踪能力（15 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 任务拆解 | 3 | 能把复杂任务拆成可执行步骤，并说明当前进度和下一步 |
| 过程记录 | 3 | 能保留关键输入、判断依据、阶段状态和用户确认点 |
| 产物沉淀 | 3 | 能形成可编辑、可复核、可归档的阶段产物和最终产物 |
| 返工处理 | 3 | 信息缺失、用户改口径或任务失败时能说明原因并继续处理 |
| 闭环建议 | 3 | 能给出后续办理、复核、整改、归档或复盘建议 |

### 5.6 易用性与业务人员采纳度（10 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 上手难度 | 2 | 非技术人员能在简短引导后独立完成任务 |
| 交互清晰度 | 2 | 产品能清楚说明需要用户补充什么、已完成什么、下一步是什么 |
| 业务语言适配 | 2 | 输出符合甲方业务语境、公文表达和部门沟通习惯 |
| 修改成本 | 2 | 业务人员二次修改量明显减少 |
| 采纳反馈 | 2 | 试用人员愿意在真实工作中继续使用相关能力 |

### 5.7 安全合规与交付适配能力（5 分）

| 二级指标 | 分值 | 评价口径 |
| --- | ---: | --- |
| 终端与环境适配 | 1 | 符合甲方办公终端、浏览器、操作系统和安全管控要求 |
| 数据安全说明 | 1 | 能说明数据存储、传输、外部调用和敏感资料处理方式 |
| 权限与审计 | 1 | 具备用户权限、操作记录、日志留存和必要审计能力 |
| 诊断与恢复 | 1 | 出现异常时便于定位问题、导出诊断信息和恢复使用 |
| 运维持续性 | 1 | 具备安装、升级、配置、备份、停用和支持服务方案 |

## 七、试用任务包设计

| 任务类型 | 目标产物 | 重点观察 |
| --- | --- | --- |
| 制度问答与依据检索 | 带依据的问答结果、待核验事项 | 是否引用准确、边界清楚、不编造 |
| 日常办公文档生成 | 纪要、报告、请示、方案、整改清单 | 是否结构规范、语气合适、少改可用 |
| 典型流程表单审核 | 审核意见、缺项清单、风险提示 | 是否识别字段、角色、流程和风险 |
| 操作步骤或办理流程校核 | 步骤校核意见、确认清单、危险点提示 | 是否理解顺序、前置条件和人工确认边界 |
| 计划编制与任务分解 | 计划初稿、人员分工、材料清单、风险措施 | 是否能形成可执行计划 |
| 缺陷或问题闭环 | 分级建议、处置流程、整改清单、复盘材料 | 是否能从描述推进到闭环处置 |

## 八、状态边界

本文件为评价体系建议稿。涉及具体制度、流程、表单、系统接口、数据分类分级和安全要求时，应以甲方现行制度、脱敏样例材料和信息安全要求为准。

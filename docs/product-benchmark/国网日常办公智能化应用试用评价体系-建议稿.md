# 国网日常办公智能化应用试用评价体系（建议稿）

版本：v0.4  
日期：2026-06-12  
适用范围：日常办公与典型业务流程智能化应用试用、评估和选型

## 一、编制说明

本评价体系用于指导甲方在试用日常办公智能化应用时，形成统一、可复核、可比较的评价口径。评价重点不放在单轮问答效果、模型参数或厂商演示效果上，而是放在产品能否进入真实办公和业务办理场景，把制度、材料、表单、流程和审批边界转化为可采纳、可复核、可持续复用的业务成果。

本体系保持技术路线中立，不限定具体厂商、模型、部署方式或实现框架。参评产品可以通过知识库、业务技能、能力模板、工具接入、流程编排、本地化交付、云端服务或混合方式实现，但最终评价以统一试用任务、业务人员真实操作、输出产物质量、量化过程证据和安全合规结果为准。

本版本根据领导反馈，进一步弱化技术实现描述，强化日常办公业务、典型业务表单、办公文档格式、可信依据、功能竞争力和可量化试用效果评价。技术量化指标只作为真实试用效果的补充，不作为单独的技术压测或模型参数比拼。

## 二、评价目标

评价日常办公业务办理能力：判断产品是否能辅助完成制度问答、会议纪要、汇报材料、请示方案、整改清单等高频办公任务。

评价典型业务流程适配能力：判断产品是否理解作业类表单、流程节点、角色责任、风险点和人工确认边界。

评价办公文档产物质量：判断产品输出是否符合甲方常用文档格式、公文表达、表格结构和归档修改要求。

评价可信与可控程度：判断产品是否能基于材料作答、说明依据、标注待核验事项，并避免把推测写成事实。

评价真实采纳价值：判断业务人员是否愿意使用，是否减少检索、整理、编写、审核和二次修改成本。

评价试用效率和稳定性：判断产品在同一任务、同一材料、同一评分标准下，是否具备可接受的有效完成时间、一次可用率、失败重试控制和成本可控性。

## 三、评价对象与边界

评价对象为参与甲方试用的日常办公智能化应用，包括企业知识问答、业务材料生成、流程表单审核、任务办理辅助、工具调用、文档整理和多步骤工作台类产品。

本体系不评价厂商宣传材料中的未试用能力，不以厂商自述功能作为直接得分依据。所有核心得分应来自统一材料、统一任务、业务人员真实操作和留存证据。

涉及工作票、操作票、检修计划、缺陷处理、制度问答等具体场景时，应由甲方提供脱敏样例材料，并以甲方现行制度、流程文件、终端环境和信息安全要求为准。

效率、稳定性和成本类指标应在输出质量达标的前提下评价。生成速度快但结果不可用、依据不清或需要大量返工的，不应仅因响应快而获得高分。

## 四、评价原则

| 原则 | 含义 | 执行要求 |
| --- | --- | --- |
| 功能可用优先 | 以业务人员真实任务完成质量作为主要评价对象 | 避免只看开放式闲聊、模型参数或厂商演示 |
| 统一材料试用 | 所有产品使用同一批脱敏材料和任务说明 | 统一输入、统一周期、统一评分表 |
| 产物可采纳 | 评价输出能否直接或少量修改后进入业务使用 | 保留原始输出、修改点、采纳情况和节省时间 |
| 依据可核验 | 关键结论应说明材料来源或制度依据 | 无法确认事项必须标注待核验 |
| 量化口径一致 | 耗时、修改次数、一次通过和失败重试等数据应按同一口径采集 | 同一任务、同一材料、同一人员或同类人员条件下比较 |
| 安全边界清楚 | 试用过程不得突破甲方数据和权限边界 | 涉敏材料脱敏，外部调用和资料留存边界清楚 |
| 人工责任保留 | 产品输出不能替代审批、审核和安全责任 | 高风险结论必须保留人工确认提示 |

## 五、评价指标体系

满分 100 分。建议将功能可用、业务产物和真实采纳相关指标作为主体，将安全合规、效率稳定性与成本可控作为支撑指标，避免评价被具体实现方式或模型参数牵引。

下表为一级指标总览，第六章按本表顺序逐项展开；二级指标名称、分值和评分口径以第六章为准。

| 一级指标 | 权重 | 二级评价重点 | 主要证据 |
| --- | ---: | --- | --- |
| 日常办公业务办理能力 | 20 | 制度问答与政策检索、会议纪要与记录整理、汇报与报告材料生成、请示方案与说明材料、整改清单与跟踪事项、办公流程辅助 | 办公任务输入输出、业务人员修改点、采纳情况 |
| 典型业务表单与流程适配能力 | 20 | 流程表单理解、字段完整性校核、角色责任识别、流程节点与前后置条件、风险与合规提示、人工确认边界 | 样例表单审核意见、缺项清单、风险提示、人工确认记录 |
| 办公文档生成与格式规范能力 | 15 | 文档结构完整、格式规范性、公文与业务语气、表格与清单产物、少量修改可用 | 文档初稿、格式检查、人工修改记录 |
| 可信依据与编造控制能力 | 15 | 引用与依据标注、事实一致性、待核验表达、编造控制、敏感信息处理 | 引用来源、错误记录、待核验事项、风险提示 |
| 任务闭环与过程可追踪能力 | 12 | 任务拆解、阶段状态与过程记录、产物沉淀、返工处理、闭环建议 | 阶段记录、最终产物、用户确认点、重做记录 |
| 易用性与业务人员采纳度 | 8 | 上手难度、交互清晰度、业务语言适配、修改成本、采纳反馈 | 试用人员评分、节省时间、满意度和采纳反馈 |
| 安全合规与交付适配能力 | 5 | 终端与环境适配、数据安全说明、权限与审计、诊断与恢复、运维持续性 | 安全说明、部署说明、诊断材料、运维记录 |
| 试用效率、稳定性与成本可控能力 | 5 | 任务有效完成时间、一次可用率、失败与重试情况、资源与使用成本 | 任务计时记录、一次通过记录、失败重试记录、成本说明 |

## 七、试用任务包设计

每个参评产品至少完成 5 类任务。甲方可根据实际业务选择任务，但各产品应使用同一任务说明和同一批脱敏材料，并按统一口径记录任务耗时、一次通过、失败重试和人工修改情况。

| 任务类型 | 目标产物 | 重点观察 |
| --- | --- | --- |
| 制度问答与依据检索 | 带依据的问答结果、待核验事项 | 是否引用准确、边界清楚、不编造，是否能在合理时间内完成 |
| 日常办公文档生成 | 纪要、报告、请示、方案、整改清单 | 是否结构规范、语气合适、少改可用，人工修改耗时是否减少 |
| 典型流程表单审核 | 审核意见、缺项清单、风险提示 | 是否识别字段、角色、流程和风险，是否保留人工确认边界 |
| 操作步骤或办理流程校核 | 步骤校核意见、确认清单、危险点提示 | 是否理解顺序、前置条件和人工确认边界，是否出现遗漏或误判 |
| 计划编制与任务分解 | 计划初稿、人员分工、材料清单、风险措施 | 是否能形成可执行计划，是否能沉淀阶段产物 |
| 缺陷或问题闭环 | 分级建议、处置流程、整改清单、复盘材料 | 是否能从描述推进到闭环处置，失败或信息不足时能否继续处理 |

## 八、评价实施流程

建议试用周期为 7 至 14 天。厂商可提供必要安装、配置和使用引导，但不应代替甲方完成评分任务。

| 阶段 | 主要动作 | 输出物 |
| --- | --- | --- |
| 准备阶段 | 确定参评产品、试用人员、周期、脱敏材料、统一计时口径和评分表 | 试用方案、任务包、评分表、过程记录表 |
| 试用阶段 | 业务人员按统一任务使用产品，并保留过程证据和量化记录 | 原始输入、产品输出、依据、耗时记录、失败重试记录 |
| 评分阶段 | 业务、信息化/安全、运维人员分别评分 | 分项评分表、问题清单、量化统计表 |
| 复核阶段 | 对关键扣分项、重大风险项、得分接近项和异常耗时项复核 | 复核意见、风险说明、异常说明 |
| 结论阶段 | 形成适用场景、落地条件和后续建议 | 试用评价报告、选型建议 |

## 九、证据采集与计分方法

### 9.1 证据采集

每项任务至少留存以下证据：

1. 原始输入材料和任务说明。
2. 产品原始输出。
3. 产品引用的依据或来源。
4. 任务开始时间、首次输出时间和可采纳完成时间。
5. 业务人员人工修改点、修改次数和修改耗时。
6. 最终是否采纳，首次输出是否一次通过。
7. 是否发生失败、重试、中断、空输出或明显格式错乱。
8. 试用人员评分和简要评价。

### 9.2 统一计时与量化记录口径

| 量化字段 | 记录口径 |
| --- | --- |
| 任务开始时间 | 用户提交完整任务材料并发起任务的时间 |
| 首次输出时间 | 产品首次给出完整可阅读结果的时间 |
| 可采纳完成时间 | 输出达到少量修改可用的时间 |
| 人工修改次数 | 业务人员为形成最终可用产物进行的实质性修改轮次 |
| 人工修改耗时 | 业务人员从产品输出到最终可用稿的处理时间 |
| 是否一次通过 | 首次完整输出是否基本可用，且无需重新发起任务 |
| 是否失败或重试 | 是否出现中断、卡死、空输出、明显格式错乱或必须重新开始 |

### 9.3 评分主体建议

| 评分主体 | 建议权重 | 重点职责 |
| --- | ---: | --- |
| 业务部门评分人员 | 70% | 评价任务完成质量、产物可用性、修改成本和采纳意愿 |
| 信息化/安全评分人员 | 20% | 评价资料接入、数据安全、权限边界和审计要求 |
| 运维管理评分人员 | 10% | 评价安装配置、稳定性、诊断恢复、升级维护和支持能力 |

### 9.4 评分等级建议

| 等级 | 分数区间 | 建议解释 |
| --- | ---: | --- |
| A | 90-100 | 可优先进入试点扩大或采购论证 |
| B | 80-89 | 具备落地价值，需补齐部分场景、合规或运维条件 |
| C | 70-79 | 可在有限场景继续观察，不宜直接大范围推广 |
| D | 60-69 | 能力或落地条件不足，需完成明显改进后再评估 |
| E | 60 以下 | 暂不建议进入正式试点或选型 |

## 十、结果应用建议

最终结论不宜只给总分排名，应同时给出：

1. 各一级指标得分和主要扣分项。
2. 产品适合优先落地的日常办公和业务办理场景。
3. 不适合或需谨慎使用的业务边界。
4. 数据安全、终端环境和系统接入前置条件。
5. 有效完成时间、人工修改成本、失败重试情况和一次可用率。
6. 后续试点、扩围、采购或淘汰建议。

对于得分接近的产品，建议优先比较真实任务采纳率、人工修改成本、依据错误数量、失败重试情况和有效完成时间，而不是单纯比较回答速度、模型参数或界面观感。

## 十四、状态边界

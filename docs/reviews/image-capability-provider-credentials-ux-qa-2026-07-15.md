# 阿里百炼单 Key 图片能力前端 UX QA 报告

- 日期：2026-07-15
- 范围：设置 → 模型配置 → 阿里云百炼快速配置；识图、生图、其他平台高级配置
- 结论：**通过当前任务发布门禁**
- P0：0
- P1：0
- P2：2（第三方平台真实凭据覆盖、自动化无障碍/视觉回归）

## 1. 用户目标

用户只需要：

1. 粘贴一份阿里云百炼 API Key；
2. 最多选择一个识图模型和一个生图模型；
3. 点击一次“保存并验证”；
4. 看到识图和生图各自的真实验证结果。

Workspace ID、Region、Base URL、协议和命名凭据不进入阿里百炼主路径。其他平台和专用 API Key 继续保留在“高级图片模型”中。

## 2. 根因与影响面

### API 验证失败根因

- 百炼 API Key 是业务空间凭据，不会自行判断要调用识图还是生图；模型和能力由请求中的模型 ID 与接口决定。
- 原运行配置把中国区业务空间 Key 发往国际站兼容地址，服务返回 401 invalid_api_key。
- 识图和生图原先分别读取多套环境变量或默认凭据，旧值可能覆盖用户刚保存的 Key。

### 布局问题根因

- 1120px 窗口中，左右导航占用空间后，模型内容列只有约 365px。
- 原响应规则依据整个窗口宽度，而不是模型配置容器宽度，因此快速配置仍保持三列并被裁切。

### 影响面

- 中国区百炼业务空间 Key 的识图、生图测试和真实使用；
- 旧默认凭据与新快速配置凭据共存时的选择优先级；
- 1120px 等“窗口不小、内容列很窄”的桌面布局；
- 其他平台专用 Key 的高级配置入口。

## 4. 真实 Electron 与阿里百炼验证

本轮在当前源码启动的真实 Electron 桌面壳中完成，使用用户授权的本机百炼凭据；报告、控制台输出和 Git 中均未记录密钥正文。

- 打开“设置 → 模型配置”，快速配置入口可见。
- Key 输入框留空，页面保留本机已有凭据。
- 识图模型：qwen3-vl-plus。
- 生图模型：qwen-image-2.0-pro。
- 点击“保存并验证”后：
  - 总状态：识图和生图均已验证；
  - 识图：已验证；
  - 生图：已验证；
  - 页面错误：0；
  - 控制台错误：0；
  - Key 输入框字符数：0。
- 生图探测真实生成并安全下载了一张测试图片，会产生少量百炼调用费用。

阿里图片结果下载仍保留 SSRF 防护：只对匹配的 DashScope OSS HTTPS/443 域名放行本机代理使用的保留地址段，不开放任意私网地址，不允许跳转泄露签名 URL。

## 5. 前端 UX 与响应式验收

### 1440px 桌面窗口

- 快速配置卡片可见；
- API Key、识图模型、生图模型同排展示；
- “保存并验证”和三段状态完整可见；
- 高级配置默认收起；
- 页面与卡片无横向溢出。

### 1120×720 窄桌面窗口

首次终验发现快速配置内容宽度 585px、可视宽度 365px，模型选择和状态条发生实际裁切，判为 P1 并阻止交付。

修正为依据 settingsPaneModels 容器宽度响应后，实时复验结果：

- 页面宽度：scrollWidth=1120，clientWidth=1120；
- 快速配置：scrollWidth=365，clientWidth=365；
- 三个输入字段改为单列，每列宽 335px；
- 状态区改为单列；
- 横向滚动消失；
- 识图、生图的已验证状态仍完整显示；
- 页面错误：0，控制台错误：0。

### 高级配置

- 默认不可见；
- 点击“高级图片模型”后 aria-expanded=true 且内容可见；
- 展开高级配置不会回显快速配置 Key。

- 未验证：项目未配置像素级视觉回归基线；截图已人工检查。

## 8. 剩余风险

- P2：其他平台/专用 API Key 的兼容路径已有配置、校验和自动化测试，但本轮没有用户提供的第三方真实 Key，因此未做第三方平台实时调用。
- P2：缺少 axe-core/Lighthouse 与自动化视觉回归基线。
- 安全提醒：用户曾在对话和截图中暴露完整临时 Key；应在验收后立即到百炼控制台重置或删除该 Key，并把新 Key 只粘贴到本机应用。

## 9. QA 结论

当前主路径满足“只提供一份百炼 API Key，最多选择两个模型，其余配置不要求用户填写”。真实 Electron 中识图和生图均已验证，窄窗口裁切问题已修复并复验，其他平台专用 Key 仍可通过高级配置使用。当前任务范围内没有 P0/P1 阻断项。

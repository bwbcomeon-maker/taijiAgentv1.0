## 1. 用户目标与信息层级

核心目标是让用户能理解并完成以下任务：

1. 阿里云百炼 API Key 是业务空间级凭据，模型由请求中的模型 ID 选择，不由 Key 自动选择。
2. 同一把百炼 Key 可以共享给识图和生图；也可以为两项能力分别覆盖独立 Key。
3. “已配置”只代表字段齐全；“已验证”必须来自真实识图或真实生图探测。
4. 外部平台专用 Key 通过平台对应的凭据字段保存；只有明确兼容 OpenAI Images 的外部图片端点才进入通用配置。

页面层级：

- 主路径：模型配置页展示主模型、看图识别、图片生成三项能力的可用性和阻断项。
- 辅助路径：提供商页管理平台凭据、外部识图 Provider 和外部图片 Provider。
- 高级项：Workspace ID、Region、Base URL、响应格式、尺寸映射、超时等按 Provider 动态显示。

## 3. 真实阿里百炼验证

使用用户提供的临时凭据，在隔离运行目录中执行最小探测；报告与代码中未记录密钥正文。

- `qwen3-vl-plus`，`cn-beijing`：真实识图探测成功，状态进入“已验证”。
- 同一配置切换到错误 Region：安全失败，返回 `vision_probe_failed`；恢复北京 Region 后重新验证成功。
- 共享凭据轮换为无效值：识图和生图状态均回退为“已配置，尚未验证”；恢复后识图重新验证成功。
- 仅轮换图片凭据：识图保持“已验证”，图片保持“已配置，尚未验证”，证明两项验证指纹相互隔离。
- `qwen-image-2.0-pro`：百炼生成请求已成功返回图片结果 URL；本机代理/DNS 将阿里 OSS 图片域名映射到保留地址 `198.18.2.13`，安全下载器按 SSRF 规则拒绝落盘。因此本次**不能判定生图端到端验证通过**，界面现会返回可行动的 `image_gen_result_url_blocked` 说明。
- 隔离运行目录已删除；临时凭据未保留在项目文件、测试产物或 Git 提交中。

## 5. 验收期间发现并修正的问题

1. 375px 和中窄桌面宽度下，待办主视图可能与设置主视图同时显示；1120–1200px 出现窄列和断字。已改为按实际配置容器宽度响应式重排。
2. “验证”按钮实际只刷新状态；外部图片字段齐全时被误标为可用；外部图片模板暗示支持尚未实现的私有协议。已修正文案、状态和入口边界。
3. 上游已生成图片但本机代理/DNS 触发安全下载拦截时，原提示过于笼统。已增加不泄露敏感信息的可行动错误码与说明。
4. 页面写明外部图片端点必须使用 HTTPS，但 HTTP 地址仍可保存。已增加前端即时校验与后端强制校验，并补充不落盘回归测试。

## 6. 可访问性与一致性

- 已实时验证：键盘可聚焦主要按钮和表单；编辑器打开后有明确焦点；错误区域使用 `aria-live`；无效字段设置 `aria-invalid` 并关联错误说明；按钮忙碌态和展开态有对应 ARIA 状态。
- 已实时验证：危险删除有确认步骤，活动 Provider 删除被阻止；取消操作保留明确的草稿放弃确认。
- 未验证：未配置 axe-core/Lighthouse 自动化，因此不能宣称自动化无障碍扫描通过。
- 未验证：项目未配置基准图视觉回归，截图已人工检查但不能宣称像素级回归通过。

## 7. 剩余风险

- P2：1440px 下图片生成配置仍位于首屏下方，需要滚动；主状态摘要可见，但完整表单发现成本仍存在。
- P2：缺少 axe-core/Lighthouse 和自动化视觉回归基线。
- P2：当前 Mac 的代理/DNS fake-IP 使阿里 OSS 结果下载被安全策略拦截；需在不削弱 SSRF 防护的前提下调整代理直连/DNS，再做一次真实生图端到端复验。

## 8. QA 结论

当前界面已满足“阿里百炼通用业务空间 Key + 各平台专用 Key”并存的配置路径，状态不再把字段齐全误报为已验证，浏览器和实际 Electron 主路径均可见、可发现、可访问。由于本机代理/DNS 阻断阿里图片结果安全下载，以及自动化无障碍/视觉回归尚未配置，本次结论为“带限制通过”，不是无条件发布通过。

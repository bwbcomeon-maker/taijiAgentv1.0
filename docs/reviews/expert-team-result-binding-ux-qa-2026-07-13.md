# 前端 UX QA 报告：专家团结果绑定与恢复

## 状态

带限制完成。未发现 P0/P1；存在自动化可访问性、窄屏和视觉回归未验证项。

## 变更范围

- 专家团工作台：启动失败、生成失败、结果待核验、历史结果未绑定、阶段成果待复核。
- Gateway 生成结果与专家团阶段结果的绑定。
- 重新核验、放弃未核验结果后重新生成及风险确认。

## 主要用户目标

用户在聊天区已经看到结果时，工作台必须准确说明该结果当前处于“已绑定、待核验、业务校验失败或历史记录无法安全绑定”中的哪一种状态，且不能误导用户重复生成。

## 主内容 / 辅助内容 / 高级内容

- 主内容：当前真实状态、下一步主操作、是否会重复调用模型。
- 辅助内容：阶段进度、协作成员、成果摘要。
- 高级内容：stream/turn、消息索引、摘要和 journal 证据；只用于后端核验，不暴露给普通用户。

## 已测试的主要用户路径

1. 旧异常 run 打开后，从“启动失败”变为“历史结果未绑定”，已有聊天结果保持可见。
2. 点击重新生成入口先出现“可能已有结果、会产生额外模型消耗”的确认提示。
3. 新 Gateway 执行完整写入 submitted、assistant_started、completed，并进入阶段成果待复核。
4. 错 stream、错 turn、错索引、摘要漂移、多候选和 completed/interrupted 冲突均拒绝认领。
5. Gateway 空回复、模型错误、用户取消、服务端取消、partial EOF 分别落到生成失败或取消，不进入待核验或成功态。

## 剩余风险与未验证项目

- 旧 run 缺少稳定外键，不能自动使用已有结果继续推进；这是防止误认领的明确安全取舍。
- `result_unverified` 未在真实 Electron 中通过故障注入单独构造，已由后端、Presenter 和前端自动化覆盖。
- 窄屏/窗口缩放、完整键盘 Tab 路径、Axe、视觉回归未验证。

## 后续建议

- 后续若要挽救旧结果，应设计“展示候选摘要—用户明确认领—记录审计事件”的人工流程，不能恢复按时间或最新消息猜测。
- 为工作台重渲染补焦点恢复与 Playwright 回归。

# 太极 Agent 麒麟终端打包交付操作说明

## 你要拷贝哪个文件夹

把整个文件夹拷贝到目标终端任意位置：

```text
taijiagent 打包交付
```

不要只拷贝里面的单个脚本。这个文件夹里包含源码包、构建脚本、安装验证脚本和校验文件。

## 目标终端已确认环境

- 系统：Kylin V10 SP1，Debian-like
- 架构：x86_64 / amd64
- 包管理器：apt、apt-get、dpkg、systemctl
- glibc：2.31

## 第一步：构建安装包

在目标终端中打开这个文件夹，右键选择“在终端中打开”，然后执行：

```bash
bash ./01_目标终端_构建安装包.sh
```

这个脚本会自动完成：

- 清理拷贝时生成的 `.DS_Store`、`._*` 等 macOS 元数据文件。
- 清理旧的 `生成的安装包/` 输出，避免失败后误装旧包。
- 校验目标终端架构和包管理器。
- 校验源码压缩包没有传输损坏。
- 解压源码到本文件夹下的 `构建工作区/`。
- 安装构建依赖。
- 安装或复用 `uv`。
- 如果系统 Node/npm 过旧，自动在本文件夹下准备隔离的 Linux x64 Node.js 构建工具。
- 生成 Linux Python venv。
- 通过 npm 获取 Linux Electron runtime。
- 构建 `amd64 .deb` 安装包。
- 把生成的安装包复制到本文件夹下的 `生成的安装包/`，并写入 `.build-success` 成功标记。

如果日志里出现下面这种内容，说明你用的是旧交付包或旧源码包，构建已经中断，不会有安装文件：

```text
The lockfile at `uv.lock` needs to be updated, but `--locked` was provided.
```

处理方式：重新拷贝当前这份新版交付文件夹，再重新执行第一步；新版脚本会自动清理拷贝元数据，并按 `SHA256SUMS.txt` 指定的源码包构建。

构建成功后，你会看到：

```text
生成的安装包/taiji-agent_0.1.0_amd64.deb
生成的安装包/taiji-agent_0.1.0_amd64.deb.sha256
```

## 第二步：安装并验证

仍然在同一个文件夹中执行：

```bash
bash ./02_目标终端_安装并验证.sh
```

这个脚本会自动完成：

- 检查第一步写入的 `.build-success` 成功标记；没有成功标记不会安装旧包。
- 校验 `.deb` 文件。
- 如果检测到旧的浏览器 WebUI 版 `taiji-agent`，会先记录旧服务状态，并停止旧服务和旧进程，让旧运行数据不再继续写入。
- 停止旧运行态后，再把旧 `/opt/taiji-agent`、系统配置和旧服务文件备份到 `旧版备份/`。
- 如果备份时遇到文件仍在变化，脚本会自动重试一次；如果最终备份失败，会保留旧安装，不会卸载、不删除、不安装新版，并尽量恢复原本正在运行的旧服务。
- 备份成功后，自动禁用旧的 `taiji-agent-webui.service` 和 `taiji-agent-gateway.service`。
- 自动清理旧包状态、旧安装目录、旧系统入口和旧桌面入口；不需要手动卸载或删除文件。
- 检查 `8787`、`18642`、`18787` 端口，遇到非太极 Agent 进程占用会停止安装并打印诊断，避免误杀其他软件。
- 使用 `sudo apt-get install` 安装太极 Agent。
- 如果机器上已经装过 `taiji-agent`，脚本会自动用当前生成的安装包覆盖安装；遇到同版本重装、apt 判定为“降级”或包被 hold，也不需要手动卸载或删除旧文件。
- 运行安装态诊断命令。
- 在图形桌面会话中尝试 Electron smoke test。
- 检查 `taiji` 命令入口。

旧版备份文件类似：

```text
旧版备份/taiji-agent-legacy-20260609_120000.tar.gz
```

这个备份可能包含模型 API Key、微信 token 或历史会话，只能留在目标机本地排障使用，不要发到群里或上传。

## 第三步：双击启动

安装验证脚本通过后，从开始菜单搜索并打开：

```text
太极 Agent
```

验收顺序：

1. 双击“太极 Agent”能打开窗口。
2. 首屏进入对话界面。
3. 再双击一次只聚焦已有窗口，不新开抢端口。
4. 关闭窗口后，本次本地会话停止。
5. 再次双击能重新打开。
6. 首次在界面里配置模型后，发送一句测试消息，确认真实对话返回。

## 重要边界

- 安装包不内置模型 API Key、微信 token、企业微信 Secret、服务器地址或私钥。
- 新版安装后只保留开始菜单“太极 Agent”桌面入口；旧浏览器访问版不会作为并存产品保留。
- 未配置模型前，只能验证桌面壳、本地 Agent/WebUI 和安装链路，不能算真实模型对话已完成。
- 目标终端构建阶段需要能访问系统软件源、npm/Electron 镜像和 Python 包镜像。脚本默认使用国内镜像；如果目标终端完全离线，需要另做离线依赖包。

## 日志位置

如果失败，把下面目录里的日志保留下来：

```text
构建日志/
```

安装后的运行诊断日志在目标机：

```text
~/.local/state/taiji-agent/logs/
```

# 通用方案模板 JSON 生成要求

你需要把用户已有成果物整理成符合 `schema.json` 的 JSON，只输出 JSON，不输出解释文字。

## 字段要求

- `cover`：填写标题、客户单位、编制单位、版本、密级、日期等封面信息。
- `chapters`：按方案逻辑拆成章节、小节、段落，段落必须放入 `paragraphs[].text`。
- `tables`：必须至少输出 2 张表，并优先来自源成果物已有的 Markdown 表格或可明确抽取的结构化表格。模板套用阶段不得凭空补表；源成果物缺表时应回到初稿生成阶段补齐。
- `figures`：必须至少输出 1 个图题和说明，并对应源成果物已有的架构图、流程图、用例图、实施路线图或图示设计块。
- `images`：必须至少输出 1 张可插入 DOCX 的图片。若归一化来源中包含 `imageReferences[].path`，必须把对应路径原样写入 `images[].path`，并填写标题、说明、宽度、高度和是否必填。若 `imageReferences[]` 同时包含 `anchorText`，必须把它写入 `images[].anchor`，用于把图片插回原文对应正文位置。源文件有图但 JSON 缺少 `images[]` 属于无效输出；源文件无图时，不要在模板阶段生成替代图片，应回到富内容初稿生成阶段补齐。
- `conclusion`：整理方案结论或实施价值。

## 输出约束

- 只输出一个合法 JSON 对象。
- 不要输出 Markdown 代码块。
- 不要保留模板占位符。
- 缺失信息可用合理的中性表述补齐，但不要编造具体数据。
